## Проект 4: AI Telegram-бот с базой знаний (FAQ + RAG)

### 📊 Анализ рынка (Freelancehunt)

Яркие примеры:

- **«Telegram bot + CHAT GPT 4 (Knowledge Base)»** (`1368419`, 08.05.2024) — с тремя ролями пользователей и эскалацией к админу
- **«Чат-бот Телеграмм + Чат GPT»** (`1235185`) — с голосовыми запросами и обучением на своей базе
- **«AI ассистент в Telegram на n8n»** (`1487808`, 11.03.2025) — явно через **RAG-метод**
- **«Впровадження AI асистента у Bitrix24»** (`1603402`, 15.02.2026) — с полным стеком **n8n + Pinecone/pgvector + Gemini Pro через OpenRouter + Memory node**

**Средний бюджет:** медиана **$700–900 (~30 000 грн)**, минимум $150–300 за бот на готовой платформе (SendPulse + OpenAI Assistant), максимум $2000–5000 за мультиканальность с CRM. Часовая ставка **$25–30**.

**Что чаще всего просят:** бот отвечает на вопросы клиентов на основе загруженной базы (PDF/DOCX/XLSX/сайт/Notion), сохраняет контекст 15–30 сообщений, при неуверенности (*«confidence low»*) **передаёт диалог менеджеру** (сообщение в отдельный чат с кнопкой «подхватить»), поддерживает голосовой ввод (Whisper), иногда 2–8 языков.

### 🎯 Цель проекта

Заменить **60–80% входящих типовых запросов** клиентам на AI-ассистента, который говорит голосом компании, не выдумывает фактов (RAG) и умно отдаёт менеджеру только реально сложные вопросы.

### ⚙️ Технический стек

- **Python 3.11 + aiogram 3.x** (или чистый n8n как альтернатива — для портфолио стоит сделать n8n-версию, так как она короче и визуально эффектна)
- **OpenAI API:** `gpt-4o-mini` для ответов ($0.15/M токенов), `text-embedding-3-small` для эмбеддингов, **или** Claude 3.5 Haiku через Anthropic
- **Chroma** (локальная векторная БД, SQLite-based, бесплатно) или **Supabase + pgvector** (free tier 500 МБ)
- Для голоса — **OpenAI Whisper API**
- **Деплой** — Railway free (Python-процесс + persistent volume для Chroma)

### 📄 Функциональные требования

Бот стартует приветствием с описанием, что умеет. На любое текстовое сообщение:

1. Эмбеддинг вопроса → **similarity search** в Chroma (топ-5 чанков)
2. Prompt в GPT-4o-mini с системным сообщением: *«Отвечай только на основе контекста. Если контекста нет — честно скажи, что передашь вопрос менеджеру. Не выдумывай»*
3. Ответ пользователю

**Память диалога** — последние 20 сообщений в Redis/SQLite per user.

**Эскалация:** если GPT возвращает флаг `needs_human: true` или confidence < 0.6 (через log-probs либо second-call-вопрос) — диалог пересылается в Telegram-чат менеджеров с кнопками `Взять диалог` / `Ответ предложить`. После «взятия» бот отключается для этого пользователя на 24 часа.

**Voice-ввод:** любые voice/audio пропускаются через Whisper и обрабатываются как текст.

**Админская команда:**
- `/upload` — принимает PDF/DOCX/TXT, чанкует (~500 токенов с overlap 50), эмбедит и добавляет в Chroma
- `/sources` — показывает список документов в базе
- `/delete <id>` — удаляет

**Оценка ответа пользователем** — inline-кнопки 👍 / 👎 под каждым ответом, результат в лог `feedback.csv`.

### 🔌 Интеграции и API

- **OpenAI:** platform.openai.com, бесплатные $5 кредит (если новый аккаунт в 2025–2026 — уточнить актуальность)
- **Telegram Bot API**
- **Chroma** — локально, не требует ключей
- **Альтернатива** — Pinecone free tier (100k векторов, 1 индекс бесплатно)
- **Railway** с persistent volume 1 ГБ free для Chroma
- **Whisper API** — тот же ключ OpenAI

### 🗂 Структура данных

- **Chroma collection `knowledge_base`:** `id`, `text`, `embedding`, `metadata={source, page, chunk_index, uploaded_at}`
- **SQLite `conversations.db`:** таблица `messages(user_id, role, content, timestamp)` и `escalations(user_id, question, status, manager_id, resolved_at)`
- **CSV `feedback.csv`:** `timestamp`, `user_id`, `question`, `answer`, `rating`, `sources`
- Папка `sources/` хранит оригиналы для пере-ингеста

### 🚀 Этапы разработки

1. Первая итерация — «echo-бот на aiogram 3» с OpenAI без RAG (просто GPT-4o-mini отвечает на вопросы)
2. Chroma: скрипт `ingest.py`, который берёт один PDF, чанкует, эмбедит и сохраняет
3. Подключить retrieval в pipeline ответа, показать, что отвечает по тексту документа
4. Память диалога (20 последних сообщений в SQLite)
5. Эскалация к менеджеру (если в ответе LLM есть тег `<escalate>` — пересылка)
6. Voice через Whisper (handler `message.voice`)
7. Админ-команды `/upload` и `/sources`
8. Фидбек-кнопки, логирование, README
9. Развернуть на Railway с mounted volume для `data/chroma/`

### ✅ Критерии готовности

- База знаний из 3 PDF (≥30 страниц всего) загружается командой `/upload` за **<2 минут**
- Бот отвечает на вопрос по существу базы за **<4 секунды**, цитируя источник
- На вне-базный вопрос (*«ты какая модель?»*) честно говорит «не знаю, передаю менеджеру» и шлёт в чат менеджеров кнопку «Взять»
- Voice распознаётся корректно на русском и украинском
- Стоимость 100 диалогов **≤ $0.20**
- В README видео-демо на 90 сек и пример `.env`

### 💡 WOW-функции для портфолио

- **Hybrid search:** параллельно с векторным поиском работает BM25 (через `rank_bm25`), результаты объединяются через **Reciprocal Rank Fusion**, что даёт **+15–20% точности** на запросах с редкими терминами/артикулами
- **Автоматическое дообучение:** когда менеджер закрывает эскалацию с ответом — бот предлагает «сохранить как FAQ?», одним кликом ответ менеджера добавляется в базу знаний как новый чанк с высоким приоритетом
- **Живая аналитика через Google Sheets:** все диалоги с их фидбеком пишутся во внешний Sheet, а на отдельной вкладке — формулы `COUNTIF(feedback=👎)/COUNTA()` показывают процент проблемных ответов и топ-10 вопросов без хорошего ответа — прямая подсказка, что дописывать в базу
